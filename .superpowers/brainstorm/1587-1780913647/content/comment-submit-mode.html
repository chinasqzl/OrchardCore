评论提交方式

评论的提交交互方式影响用户体验和前端复杂度

A

传统表单提交

评论表单在页面底部，提交后整页刷新。最简单可靠，符合 OrchardCore 标准 MVC 模式，无需前端框架。

AJAX 无刷新提交

评论通过 fetch API 提交，提交后动态追加到列表，无需刷新页面。体验流畅，需要前端 JS 和 API 端点。

渐进增强（推荐）

默认使用传统表单提交（保证无 JS 也能用），当 JS 可用时自动升级为 AJAX 提交。兼容性最好，符合 OrchardCore 哲学。

无 JS 时
表单 POST → 页面刷新 → 显示新评论
有 JS 时
fetch POST → 动态追加 → 无刷新
原理：表单正常提交，JS 拦截 submit 事件 → e.preventDefault() → fetch 提交 → 动态更新 DOM
